ASTROSTDS-1116 Done Improvement

Need a way to specify length of fields in character arrays

Need a way to specify length of fields in character arrays.  For example, the SpVecDataToArray function has a "xs_spVec" parameter that is 512 characters long.   

"XS_SPVEC_SECCLASS_1" is 0, which is the Security Classification, but there's no way for users to know how many positions in the xs_spVec array is used.   Perhaps they are to assume the "_1" suffix in the name says its 1 character long?

"XS_SPVEC_SATNAME_8" is 1, which is the Satellite Common Name, but there's no way for users to know how many positions in the xs_spVec array is used.   Perhaps they are to assume the "_8" suffix in the name says its 8 characters long?   But that's not correct because it actually goes from position 1 to 8, which is 7 characters long.

 

The naming convention for string fields in a character array should look like this:

XS_FIELDNAME_STARTINDEX_LENGH = STARTINDEX

XS_FIELDNAME_LENGTH = length of the whole string

startIndex: The zero-based starting character position of a substring in this instance

length: The number of characters in the substring

 

For example:

XS_SEN_SECCLASS_0_1: security class start index is 0 and its length is 1 character

XS_SEN_ORB_BSVEC1_27_1: orbiting sensor's boresight vector #1, start index is 27 and its length is 1 character

XS_SEN_LENGTH = 512
