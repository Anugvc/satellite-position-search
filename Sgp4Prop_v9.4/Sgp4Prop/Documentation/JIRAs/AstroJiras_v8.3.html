ASTROSTDS-746 Done New Feature

Create application to convert JNA source files to JNI source files

This is to make coding faster.  Just need to write JNA code, then run the conversion, instead of tedious search & replace.

To convert JNA source code to JNI, the following tasks need to be done (in this order):
* Capture variable name assigned to instance, then replace "new IntByReference()" to "new int [1]"
* Capture variable name assigned to instance, then replace "new DoubleByReference()" to "new double [1]"
* Capture variable name assigned to instance, then replace "new LongByReference()" to "new long [1]"
* Capture variable name assigned to instance, then replace "new ByteByReference()" to "new byte [1]"
* Replace "IntByReference" to "int[]" (these are arguments passed to methods)
* Replace "DoubleByReference" to "double[]"
* Replace "LongByReference" to "long[]"
* Replace "ByteByReference" to "byte[]"
* Replace "Jna" with "Jni"
* Replace "JNA" with "JNI" (affects comments)
* Any variable captured that has ".getValue()" appended to it, replace ".getValue()" with "[0]"
* Any variable captured that has ".setValue(<arg>)" appended to it, replace ".setValue(<arg>)" with "[0]=<arg>"
* Delete all lines that start with "import com.sun.jna"

ASTROSTDS-672 Done Improvement

investigate sincos vs sin & cos

Intel C compiler has a function called sincos that simultaneously computes the sin and cos of an angle.  Without optimizations, it is about 30% faster than calling sin and cos separately.  There are several places where we call sin and cos of the same angle. 

 

Reportedly, -O2 and -O3 optimizations will use sincos automatically.  Might even need the -fast option.

 

After investigation, it appears that calling sin and cos separately through Fortran is faster than calling sin and cos through C but not sincos.  Speedup seems to be about 30-40%

When Tom ran it, though, everything seemed to slow down by a very small amount.  Could be a compiler optimization thing.
